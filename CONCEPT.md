## 0. ЧТО ЭТО
Киберпанк auto/idle-RPG про отряд киборгов-бойцов. Собираешь команду, наращиваешь её кибернетическим железом (импланты), бой идёт авто/полу-авто, ты оптимизируешь сборку. Сеттинг — киберпанк (НЕ фэнтези: сознательный уход от 90% жанра-болота, и любимая эстетика Рамиля).

## 1. РЫНОЧНЫЙ КОНТЕКСТ (честно, 20.06.2026)
- 🚨 Жанр auto/idle-RPG ПЕРЕПОЛНЕН. Мобильный F2P-gacha-сегмент = самый жестокий для соло: решает маркетинг-бюджет (user acquisition), не геймплей. Соло без UA-бюджета там не виден.
- ✅ НО соло реально выстреливали в idle-RPG через **premium на Steam (+ порт на мобилу), нишево, на ОДНОЙ свежей механике**: Melvor Idle (соло, сотни тысяч), NGU Idle, Cookie Clicker, Idle Slayer.
- **Вывод-склонность:** делать как premium / честную игру (Steam-first или premium-mobile), НЕ как гача-доилку. Фишка обязательна — иначе «ещё одна авто-RPG».
- **ПЛАТФОРМА — ОТКРЫТО:** mobile F2P vs premium Steam(+mobile). Склонность — premium (там у соло шанс).

## 2. ФИШКА (ядро отличия) — ПРОКАЧКА ЧЕРЕЗ КИБЕР-ИМПЛАНТЫ
Прогрессия не «+1 уровень», а **вживляешь железо в слоты тела**. Киберпанк — КОРЕНЬ механики, а не обои.
- Видно на модели (вставил бронелицо танку — он меняется) → прогрессия = косметика = читаемость билда с одного взгляда.
- Глубина билдов без доната. Импланты падают с боёв / крафт / апгрейд по редкости.
- Перенос идеи из Movement Parkour Racer (там протагонист = модульное кибер-тело, сменный корпус) — в паркуре было сложно, тут в самый раз.
- **Крючок №2:** импланты дают сет-синергии между бойцами → авто-бой выигрывает умная СБОРКА отряда+железа, не задрот-гринд (анти-p2w).

## 3. КЛАССЫ (4, классическая ролевая четвёрка + сигнатурный слот)
- 🎯 **Снайпер** — дальний бурст. Сигнатура: ГЛАЗА (оптика, крит, дальнобой).
- 🔫 **Штурмовик** — ближний/средний дамаг. Сигнатура: РУКИ (оружейные модули, шквал).
- 🦾 **Танк** — кибер-гигант, держит удар. Сигнатура: ТОРС + БРОНЕЛИЦО (реактор, пластины).
- 💻 **Хакер/Нетраннер** — контроль/саппорт. Сигнатура: НЕЙРОИМПЛАНТ/голова (дроны, взломы, дебаффы).
4 контрастных силуэта (тощий снайпер / боец / гора-танк / худой хакер с дронами) = читаемо и стильно. Кросс-билды разрешены (снайпер с боевыми руками = гибрид).

## 4. СЛОИ ПРОКАЧКИ (глубина = несколько систем с РАЗНЫМ темпом — секрет жанра)
Принцип: всегда в 2 минутах от апгрейда И всегда есть далёкая цель.
1. **🔼 Уровень бойца** — база (HP/урон), за бой. Быстрая морковка.
2. **🦾 Импланты** — железо в слоты тела (глаза/руки/торс/нейро/ноги). БИЛД. По редкости (обычный→легендарный).
3. **💾 Прошивка имплантов (софт)** — каждый имплант сам качается. Микро-морковка ВНУТРИ железа.
4. **🧬 Шасси / ранг (эволюция)** — дубликаты бойца → апгрейд корпуса, выше редкость, новый облик. Медленный престиж. Гача-механика БЕЗ доната.
5. **🌳 Дерево аугментаций класса** — ветки специализации (снайпер: крит/дальнобой/дебаффы; танк: броня/контратака/провокация). Идентичность билда.
6. **⚙️ Оружие** — отдельная ось сборки (пушки/клинки/дрон-пушки).
7. **🏢 Мета: ШТАБ / кибердек** — баффы всему отряду + idle-доход офлайн (сердце idle-петли). Аккаунт-прогрессия, самая долгая.

## 4Б. БОЁВКА — авто-файтер со СКИЛЛ-КЛАПАНОМ (решено 20.06.2026)
Классический авто-файтер: бойцы идут и дерутся сами. Игрок прожимает **ульты** (активные способности). НО на боссах добавляется активный слой — **клик-боксы**.
**Клик-боксы (на боссах, короткие окна — успей нажать):**
- 🎯 **Урон-окно** — попал в тайминг → крит/бонус-залп.
- 🛡️ **Уворот/блок-окно** — успел → команда уклоняется от телеги босса; проспал → урон.
- ⚡ **Заряд ульты** — нажатия копят/ускоряют ульту.
- 💻 **Класс-окна** — хакер: «взлом» (вырубить атаку босса); снайпер: «прицел» (отметить уязвимое место). Импланты УЛУЧШАЮТ эти окна → прямая связь боёвки с прокачкой §4.
**ЗАЧЕМ (ядро, решает болезнь жанра):** большинство авто-RPG = чистый пассив → стенку пробиваешь только донатом/задротом. Скилл-клапан даёт пройти босса РУКАМИ, когда на автомате уже не идёт. → анти-p2w (честность как в паркуре: пробиваешь скиллом, не кошельком) + отличие от дженерика (в авто-RPG скилл нигде не значит ничего) + вовлечение (босс = активный данс, не «смотрю бой»).
**🚨 БАЛАНС-ПРИНЦИП:** авто проходит рутину/фарм; боссы = стена (недокач + автомат = не пройти), НО активной игрой можно КОМПЕНСИРОВАТЬ недокач. Скилл = ШАНС, не гарантия (не бог при нуле кача, но обгоняешь свою прокачку). Казуал догонит гриндом, скилловый пробьёт раньше руками — никого не выгоняем, донат не нужен. Соотношение «сколько скилл компенсирует» — крутить.
**Платформа-бонус:** клик-боксы одинаково работают на тач (мобила) и ПК → не привязывает к платформе.

## 8. ДВИЖОК И ИНСТРУМЕНТЫ (решено 20.06.2026, по ресёрчу vibe coding)
**ДВИЖОК — GODOT 4.x (GDScript; C# в кармане для сложной логики).** Кодит AI-агент (Claude Code) → выбор оптимизирован под VIBE-КОДИНГ.
- **Почему Godot (для вайб-кодинга «правка→проверка» сотни раз — трение на каждой итерации минимально):**
  - Всё текстовое (.gd/.tscn/.tres) → агент правит файлы НАПРЯМУЮ, без MCP-костылей (в Unity сцены = YAML+GUID, руками не править).
  - Один навязанный паттерн (сигналы+дерево сцен) → агент меньше «фантазирует» (в Unity много валидных путей → угадайка).
  - Мгновенный рестарт, нативные логи, НЕТ domain reload (в Unity тормозит цикл на масштабе).
  - Бесплатно навсегда (MIT, 0 роялти), 2D — родная стихия.
- **Почему НЕ Unity:** C# нейросети знают лучше GDScript (плюс Unity), MCP зрелее (CoplayDev 10.9k★) — НО минус GDScript средний и сокращается на моделях 2026, а Unity проигрывает на всём остальном цикле (GUID-сцены, domain reload, лицензия Personal-лимит/fee-история).
- **Почему НЕ Unreal:** тяжёл/неудобен для idle/UI/мобилки (бинарные ассеты, жирный мобильный билд).
- 🔑 **Митигейшн единственного минуса Godot:** упрёмся в качество GDScript-генерации → пишем критичную логику на **C# внутри Godot** (.NET), сохраняя текстовые сцены и лёгкий цикл. «Лучшее из двух» без смены движка.

**ИНСТРУМЕНТЫ ДЛЯ CLAUDE CODE:**
- **База (без MCP):** прямое редактирование текстовых файлов + headless-CLI (`godot --headless`: запуск сцен, `-s script.gd`, экспорт, импорт). Полный цикл «правка → headless-прогон → читаю ошибки».
- **Тесты:** GUT (bitwes/Gut, MIT, GDScript) — headless из CLI, exit 0/1 для гейта. Агент пишет И прогоняет сам. (Если C# — gdUnit4, MIT.)
- **MCP (опционально, для скринов/захвата ошибок):** Coding-Solo/godot-mcp (MIT, ~3.9k★) или GoPeak/HaD0Yun (MIT, 95+ тулзов) или IvanMurzak/Godot-MCP (Apache-2.0, C#-аддон). Все коммерчески чисты. GDAI — проприетарный, НЕ вендорить.
- ⚠️ **Подводные камни:** `.uid`-сайдкары (Godot 4.4+) КОММИТИТЬ в git (не в .gitignore), руками не выдумывать; CLI молча глотает опечатки во флагах; на свежем чекауте сначала прогрев `--headless --import`, потом экспорт/тесты.
