# Бесконечная прогрессия — консолидированный план (2 ресёрча + мат-критик читал код)

Для перепроверки в другой нейронке. Сведено из: общего ресёрча + глубокого мульти-углового + адверсариального критика (который прочитал реальный `prototype/main.gd`).

## КОРЕНЬ ПОТОЛКА ~стадия 66 (математически, по нашему коду)
Чтобы пройти стадию `s→s+1`, нужно ×1.34 мощи (`ENEMY_HP_PER_STAGE=1.34`). Достижимая стадия = неподвижная точка где `мощь(d) ≥ 140·1.34^d`. Стадия растёт как **логарифм мощи**: `s ≈ ln(мощь)/ln(1.34)`.

**ТРИ наложенных потолка (критик нашёл, синтез сначала пропустил):**
1. 🔴 ГЛАВНЫЙ — аугменты. Реальная формула (НЕ чистый компаунд!): `aug = exp(c·totlvl^0.88)` (`AUG_DIMINISH=0.88`, `_augmul`). Показатель `totlvl^0.88` — **суб-линейный** (q=0.88<1) → растянутая экспонента → асимптотически ПРОИГРЫВАЕТ чистой экспоненте HP. Это корень.
2. 🟠 Ядра престижа `_cores_gain = 10·√depth` — **корневой** рост (sub-linear) → за N престижей ядра ∝ N, уровни аугментов ∝ log(N) → мощь ∝ (log N)^0.88. Поэтому 1500 престижей не двигают потолок.
3. 🟡 Жёсткий `STAT_CAP=1e15` — клампит урон И HP → стена ~85-100 даже если формулы починить.
+ Сингулярность `meta_pow=1+0.3·lvl` — **аддитивна** (не множит), тоже затухает.

## РЕШЕНИЕ (loop-gain теорема, сходятся оба ресёрча + AdCap/Clicker Heroes/TT2)
Вечный рост ⟺ престиж-петля с коэффициентом **αβ≥1**: валюта `C∝g^(α·глубина)`, перма-множитель `M∝C^β`. Нужно сделать рост мощи НЕ медленнее экспоненты HP.

## КОНКРЕТНЫЕ ИЗМЕНЕНИЯ В КОДЕ (приоритет)
1. **`AUG_DIMINISH: 0.88 → 1.0`** (главный фикс — делает аугменты `exp(c·totlvl)` = ИСТИННАЯ экспонента, обгоняет HP).
2. **`STAT_CAP: 1e15 → 1e300`** (снять кламп; 1e300 даёт практический потолок ~стадия 2300+ = эффективно бесконечно для казуала / 500+ часов БЕЗ риска float64-overflow 1.8e308). Поднять порог научной записи в `_fmt_n` (уже 1e15, оставить или поднять).
3. **Ядра экспоненциальные:** `_cores_gain: 10·√depth → ~4·1.16^depth` (компаунд по глубине → totlvl растёт быстро → exp(c·totlvl) взлетает).
4. **Перма-множитель от суммы ядер:** новый `(1+cores_total)^0.5` — ядро бесконечной петли.
5. **Сингулярность мультипликативной:** `meta_pow: 1+0.3·lvl → 1.5^lvl`; кванты `1.2^(stage-40)`; Memory Bonus `1.03^best_stage`.

## ПРОВЕРКА (критик)
- ⚠️ float64 рвётся на 1.8e308. Кап 1e300 = безопасно + потолок ~ст.2300 (≈бесконечно для казуала). Для ИСТИННОЙ бесконечности (>ст.2300) нужен log-пространство хранения урона/HP или break_infinity-Decimal — это бОльшая переделка, делать ТОЛЬКО если 2300 стадий мало (вряд ли).
- Цель калибровки ботами: «+несколько стадий за престиж» стабильно на горизонте тысяч престижей, бот проходит >500 стадий без потолка.
